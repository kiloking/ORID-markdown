1. 什么是管理注意力呢？

bob 用了几招管理大家的注意力。

首先

1.) 课程开场发问卷让每个人都能确立有自己的学习目标。一旦有学习目标，目的就不在于听老师怎么运课，而是会专注在自己想什么要带走什么，这样就与老师怎么上课无关。

学生会关注在自己要带走什么身上。而不是自己一边全学全记，再想运用在什么场景之内。

2. ) 为了避免大家自己剧透自己，老是想要跳掉老师的上课，一直偷看后面讲义。他干脆让大家花三分钟先看个够，满足好奇心之后，就会认真听老师讲。

3. )让大家很忙。当 bob 讲他的 90/20/8/4 原则。我才发现原来 bob 算的好好的，每 8 分钟就让我们卷入他的活动之中。难怪忙得要死。

4) 为什么让同学一直很忙参加活动。这是因为让同学有「主动掌握权」。科学原理，人一旦只要有主动权就会感到心安、积极。有目标感就会感到有动力。

于是 bob 开场的仪式都是围绕在让「大家能确立自己目标」、「无时无刻课程都跟自己有关」

这忙也不是瞎忙，而几乎是等于每 20 分钟都同组做了复述练习。这是学习法当中最高的层次。而且还是不知不觉用各种形式干了。比如说自己写小卡片，团体写大字报，找 Learing Partner，早上写记忆卡，重画 Windowpane。

2. 管理精力

人的精力是很宝贵的。所以 bob 会避免学生的精力被卷进去无谓的活动。

因此浪费学生精力的事情他会避免做。

所以他会提醒我们教学生时，千万不要让大家站起来「自我介绍，以及为什么要来上这个课」。

他说有些傻逼课，这样一搞一个小时就不见了。

拜托，没人对「别人为什么来上课」有兴趣的。

他用了巧妙的一招转换形式。他让大家自己先确认自己学习目标，再用学习卡交换。一人换三张，有效的结束。

再来，上课时有个同学问，他看课堂讨论很热烈，那老师要强制打断还继续呢？

Bob 说他会强制打断。就是在这时候他讲了最重要的一句：「上课就是围绕著精力管理」。

有些人可能觉得这段很 high ，有些人会觉得这段很无聊。但老师的职责是管理好大家精力上的使用。

3. 管理记忆力

这是我从 bob 身上学到的最牛逼之处。

很多老师学培训技巧只学到皮没学到骨。著墨于玩游戏带活动，搞得大家很 high，而不关心学生「真正」学会了多少。

综观 bob 的培训，完全是针对「如何记更好更多更牢更容易复习」奔去。

不管是提词单字卡、WindowPane、复数练习、复习时间间隔。全部都是基于科学研究设计的活动。

之前我有一阵子，研究大脑认知科学非常深，甚至还写了一本书「打造超人大脑」专注于高效学习法。

当我看到 bob 带课完全是以科学记忆为根底，教大家先学好高效学习方法再传授正确知识，简直是要兴奋的跳起来。

这是我第一次亲眼见识领悟到正宗太极拳，原来是这样打的。。。。。。。

真的，就我这一辈子。从没见过有一个老师是先教我正确学姿势，才传授我知识的。

多半是趴就丢来知识，学生学不会就是傻逼。

C 是 Content，P 是 Participate 参与，R 是 Review。20 分钟做到一个学习、参与、复习的闭环。

## 科学原理
Bob 其实花很少时间讲道理。其实更多时间是口诀、活动、讨论。

只有中间少少时间跟我们讲解当中重要的核心。

其实呢。上课最重要的就是几点

1. 管理预期、设定目标
2. 管理精力
3. 回家记得上课的内容

所以他主要的设计都是让我们怎么有目标的学会。不会大量浪费时间在无谓的地方上。

比如。他提出了一个叫 90, 20, 8 ,4 法则

90 min是一个 module 的最长长度。
20 min 是记得内容的最长长度
8 min 是学生愿意参与活动的最长长度
4 min 是学生注意力可以维持的长度

你要小心运用不要超过这些的限制。他建议我们把一个模块，最多只能 90 分钟。然后 20 分钟做一次活动设计。

这个活动设计有三个步骤。叫 C P R

C 是 Content ，Ｐ是 Participate，R 是 Revist。

20 分钟要完成一组：教他内容，让他参与，还能够温习。

我上完课能写这么多内容。不是因为我记忆多牛逼。而是他的这套教学法，让我狠狠的记得绝大多数内容。

再来。他还教我们了两招。

一招就是刚刚学的九宫格。我大概玩了两三遍九宫格，他还叫我们用九宫格设计课程重点。

我才发现九宫格就是短期记忆 7加减2的限制

一招是心智图。

上课最后他教我们怎么用心智图。我才知道我心智图以前怎么用错。

首先　他让我们写下　如果我回去要怎么吹 bob 怎么好时

## 主动讨论、Learning Partner、Partner for Learning
这个 workshop 我觉得设计非常好的一点。

就是有大量讨论环节。

有小组讨论也有两人讨论。

我觉得 Bob 控制得非常好的一个地方。就是他知道如何让讨论发挥最大效用。

有时候你是同桌要找一个人讨论交流的心得。叫 Learning Partner。然后这个双人组，对象会换来换去。两天下来基本上你整组都换过了。

有时候是三人一组。三人讨论完以后去跟其他组混合大结论。

有时候是六个人一组讨论，共同创作。

有的时候，是去让你找隔壁组的一个人，跟他双人交流。

各有不同的效果。

如果有要发表的。就是各组派一个人，只做一个结论的陈述。

后来 Bob 解释为什么要这么做。

1. 大量讨论，会让学生有「课程控制在我自己手上」的安心感，会更积极主动参与
2. 有时候会双人讨论。因为这样会比较有效率。要混著不同人，是避免同组有你不喜欢的人，还有一直换人，可以让你整组都混熟
3. 然后这种workshop 有人要 networking 有人很内向不要。所以他顶多让你去混熟 1-2　个人这样。
4. 他尽量让大家都是同组讨论，不会大混战讨论，是让大家有安心感。

然后组长设计他也是轮调的。因为这样也可以避免一个人霸占组长。

有些人一个人很霸道不让其他人玩，这样会很痛苦。

或者是一个人做得很糟糕，让学习体验很烂。

所以是轮流来。而且组长的选法也很奇琶，他利用大多数人对于当组长的恐惧，不想当组长，来催促大家的行动….

利用这种混著讨论的方式。可以时时有双人加倍学习的惊喜感。

我一直对研究如何教学以及学习这类议题非常有兴趣。

也知道「Lecture」演讲形式是所有教学形式里面最糟糕的一种。

但是怎么样做好一场教学呢？这其实是很大的学问与功夫。

首先，准备一门课。讲师，首先要对这门课的内容寥若指掌。接著要能够设计结构，让这门课的内容被精彩的呈现。

精彩的呈现，包含了

1. 管理学生的预期
2. 调动学生的注意力
3. 让学生能够产生更多创意、联想、应用
4. 课堂间可以产生有效高效的讨论
5. 产生课堂外的连结
6. 课后能够立刻应用

要能够产生这些结果非常的难。

而 Bob 大师的课，就是教你如何做到以上的每一件事！

当过老师的都知道。对自己的课能寥若指掌。并不是最难的。

最难的是同学有兴趣听你讲两天，记得所有细节。不靠任何奖励以及意犹未尽。

这就牵扯到课程如何设计。

Bob 的这堂课，并不包含结构怎么设计。而是如何运用一些结构与技巧，让课程永远充满动力。

而且这堂课中课，充分让你学到一个老师怎么样控制课程里面的动力与精力。

我举一些印象深刻的桥段设计。

## 分组设计
Bob 的开场非常不一样。他的开场是这样的。刚开始大家随便入座，然后让大家举手，听的懂英文的站教室左边，听不懂英文的站教室右边。

站好以后。他又让大家互相比自己的教学年限，教书越长的站左边，越短的站右边。

然后大家排完以后。他让两边的人串成一长列。要大家报数。1报到9，如此反覆循环。

你报到多少号去哪一组。

知道自己去哪一组之后，速度要快。最晚到新一组的人，要当组长。
（后来才知道这招很坏，利用大家都不想当组长的心理，催促大家行动。他这两天一直用这招逼我们）

坐定之后，他才跟我们说为什么要这么做。

他说，一般参加这种workshop，一定是熟人坐一桌，不想跟人接触的人坐一桌，上课想偷懒的人坐一桌，所以用这个方法可以有效平均打散大家。

## 预习课本
当我们坐定之后，他让我们小组推派一个人去跟他领课本。拿到课本 / 劳作簿以后，大家当然开始翻内容啦。

他要我们大家停下来。

要大家花 3 分钟时间，这一本课本大概80页。每个人发三张便条纸，选择自己最感兴趣的三页做记号。

做好之后。组内大家分享，自己最有兴趣哪三页，组长记下来。而且组内开始自己会讲自己为什么喜欢那三页。

正当我们在想这是要干嘛时。

Bob 解释说。通常课本发下来之后，总会有那种讨厌的学生，情不自禁的会掀翻课本，自己就剧透自己，后面就不想上了。

你让它们开场翻个够，让它们选自己喜欢的三页。自己趁机解释今天会上哪些内容。这些学生开场翻够满足好奇心后，后面他就不会一直想翻跳页了。
